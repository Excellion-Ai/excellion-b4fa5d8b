

## Fix OAuth Sign-In After Remix

### Problem
After remixing, both Google and GitHub OAuth return "Unsupported provider: provider is not enabled" (400 error). The code uses `supabase.auth.signInWithOAuth()` directly, which no longer works.

### Important Limitation
**GitHub OAuth is not supported** on Lovable Cloud. Only Google and Apple are available as OAuth providers. The GitHub sign-in button will need to be removed or replaced with Apple.

### Changes

#### Step 1: Configure Google OAuth
Use the Lovable Cloud social auth configuration tool to generate the managed OAuth module (`src/integrations/lovable/`). This enables Google sign-in without needing your own credentials.

#### Step 2: Update Auth.tsx
- Replace `supabase.auth.signInWithOAuth({ provider: "google", ... })` with `lovable.auth.signInWithOAuth("google", { redirect_uri: window.location.origin })`.
- Remove the GitHub OAuth button (or replace it with Apple sign-in if desired).
- Import the new `lovable` client from `@/integrations/lovable/index`.

### Technical Details

**Before (broken):**
```typescript
const { error } = await supabase.auth.signInWithOAuth({
  provider,
  options: { redirectTo: `${window.location.origin}${redirectTo}` },
});
```

**After (working):**
```typescript
import { lovable } from "@/integrations/lovable/index";

// Google sign-in
const { error } = await lovable.auth.signInWithOAuth("google", {
  redirect_uri: window.location.origin,
});
```

### Summary of Changes
- **src/pages/Auth.tsx** -- Switch Google OAuth to managed solution; remove GitHub button
- **src/integrations/lovable/** -- Auto-generated by the configure tool (not manually edited)
